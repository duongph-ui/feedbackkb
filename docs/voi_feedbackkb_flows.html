VOI FeedbackKB — Luồng chức năng × UX theo persona

DrawVOI-Architect · nguồn: ArchitecturePack V1.4 + codebase (routes/agents/widget/mcp/cli) · 2026-06-24

🎯 Goal (Bước 1.2)
Bày toàn bộ tương tác của FeedbackKB lên 1 mặt phẳng: persona × chức năng × stage vòng đời — để thấy đủ độ phủ, gap, và điểm UX trọng yếu cho từng loại user.
  • Người dùng: End-user (báo lỗi) · Triager · Admin/PO · Developer · Agent team (Conductor/Triage/Analyst/Fixer).
  • Nỗi đau: (1) bug-fix knowledge rò rỉ khỏi prompt lẻ; (2) hệ thống internal không có kênh feedback có cấu trúc.
  • Mục tiêu: 1 nơi vừa thu feedback (widget 1-ô) vừa tích lũy knowledge (capture-fix) → agent triage tự động + fix bán tự động, người chỉ duyệt việc quan trọng.

3 · Flow per persona + UX trọng yếu

👤 End-user Human — báo lỗi (UF1)

sequenceDiagram actor U as End-user participant W as Widget participant API as Feedback API participant DB as Postgres (fbk) U->>W: bấm nút "Phản hồi" W->>W: html2canvas chụp NGAY (trước khi mở panel) → đính sẵn U->>W: nhập 1 ô nội dung (tuỳ chọn khoanh đỏ / blur / dán ảnh) W->>API: POST /api/feedback (JWT | X-App-Key | none) API->>DB: INSERT fbk.feedback status=new + enqueue triage API-->>U: "Đã ghi nhận. Cảm ơn!"
UX trọng yếu
  • Zero-choice: chỉ 1 ô message*. Không chọn type / đặt tiêu đề / 3 link — Triage agent tự suy.
  • Auto-screenshot, local-only: mở form là đã chụp + đính sẵn, ảnh KHÔNG upload tới khi bấm Gửi (privacy).
  • Annotate nhẹ: khoanh đỏ / mũi tên / blur (làm phẳng cứng) cho dữ liệu nhạy cảm; hoặc Bỏ ảnh.
  • Phản hồi tức thì: 200 + cảm ơn ngay; mọi xử lý nặng async (không bắt user chờ).

🧑‍💻 Developer Human — tự phục vụ knowledge (UF2 + §3.4.3)

sequenceDiagram actor D as Dev (Claude Code) participant M as feedbackkb-mcp participant WK as wiki (sepo) D->>M: search_knowledge("lỗi X") M-->>D: 2 lesson cũ (kèm cách fix) Note over D: tái dùng fix cũ, KHÔNG prompt lại từ đầu D->>D: fix code D->>M: capture_lesson(symptom, root_cause, fix, files, prevent) M->>WK: put_doc → KB/system/cat/slug WK-->>D: wiki_path (knowledge tích lũy)
UX trọng yếu
  • Trong IDE, không rời flow: tra + ghi knowledge ngay trong Claude Code — mấu chốt giải nỗi đau #1.
  • Setup 1 lệnh: npx feedbackkb register / init-mcp / init-hook — không cấu hình tay, không chạm schema PG.
  • Tự động hoá: Stop-hook rút lesson cuối session (lọc nhiễu: chỉ khi có dấu hiệu fix bug thật).
  • 2 cổng I/O: REST + MCP — không bao giờ viết SQL.

🗂️ Triager Human — dashboard review

sequenceDiagram actor T as Triager participant DASH as Dashboard participant API as Feedback API T->>DASH: lọc theo system/status DASH->>API: GET /api/feedback?system=&status= (tenant-scoped) T->>DASH: click hàng → chi tiết + events + agent_tasks alt thiếu thông tin T->>API: PATCH status=needs_info else hợp lệ T->>API: PATCH status=triaged/resolved (guarded transition) end API->>API: status-machine guard + ghi feedback_event(actor=human)
UX trọng yếu
  • Danh sách đã có tiêu đề: Triage agent rút name từ message → dễ scan, không phải mở từng cái.
  • Tenant-scoped tự động: chỉ thấy feedback của system mình (không cross-tenant).
  • Transition có rào: chỉ cho đổi status hợp lệ; bug nặng resolved bắt buộc có lesson (CL2).
  • Audit minh bạch: mọi đổi trạng thái ghi append-only, xem được lịch sử ai làm gì.

🛡️ Admin/PO Human — gate quyết định (UF3) + quản trị

sequenceDiagram participant F as Fixer agent participant DB as Postgres actor A as Admin/PO participant PR as Repo PR F->>DB: agent_task(stage=fix, decision_needed=true) F->>A: "Apply fix [X] lên prod? severity=high" alt approve A->>F: yes → ghi feedback_event(actor=human, action=approve) F->>PR: tạo PR draft F->>DB: status=in_progress else reject A->>F: no / chỉnh hướng F->>DB: status=wontfix | re-investigate end
UX trọng yếu
  • Chỉ duyệt việc irreversible: apply prod / merge PR / xoá lesson — agent KHÔNG tự làm kể cả severity=crit.
  • Quyết định 1 click: approve/reject ngay tại câu hỏi, có đủ ngữ cảnh (fix đề xuất + severity).
  • Quản trị gọn: register/rotate key, GDPR export/delete/erase, config privacy/retention — qua admin route.
  • Onboard không cấp gì: seed env / open-register cho consumer chạy mà không phải cấp key thủ công.

🤖 Agent team Machine — Conductor điều phối (§3.3)

flowchart LR Q[(hàng đợi: feedback new)] --> CO{Conductor} CO -->|stage=triage| TR[Triage: classify + dedupe] TR -->|need info| NI[needs_info → hỏi user] TR -->|hợp lệ| AN[Analyst: root-cause + ground knowledge] AN --> FX[Fixer: propose fix + draft lesson] FX -->|decision_needed=true| GATE{{⛔ Human gate}} GATE -->|approve| AP[apply / PR · status=in_progress] GATE -->|reject| WF[wontfix / re-investigate]
UX/Design trọng yếu (cho AI consumer)
  • Least-privilege: mỗi agent là subagent .claude/agents/fbk-*.md với tool tối thiểu cho stage của nó.
  • Untrusted boundary: feedback/ảnh OCR/repo ngoài bọc delimiter — KHÔNG để "ra lệnh" cho agent (chống prompt-injection).
  • Queue an toàn: claim task bằng FOR UPDATE SKIP LOCKED + idempotency_key → không double-process.
  • Hard stop: Fixer luôn dừng ở gate; con người là người duy nhất chạm prod.
